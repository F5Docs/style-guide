# Contractions

## Guidance

Use contractions. They make content sound like a human wrote it, which
is the goal. Only avoid a contraction when it creates ambiguity or
reads awkwardly aloud.

## Examples

**Do:**
> If the system doesn't respond, restart the service.

> You'll need to set up a virtual server before continuing.

**Don't:**
> If the system does not respond, it will be necessary to restart the service.

## Notes

Common contractions to use freely: don't, can't, won't, you'll, let's,
we're, it's, there's, that's, isn't, aren't.

Test by reading the sentence aloud. If the contraction sounds natural
to a native English speaker, use it. If it sounds forced or ambiguous
(e.g., "there'd"), spell it out.

"Y'all" and similar regional contractions are not appropriate in
product documentation.
