# Second person

## Guidance

Write from the user's perspective using "you" and "your." Avoid
third-person references like "the user" or "the administrator" unless
necessary to avoid confusion between multiple actors.

## Examples

**Do:**
> You can set up rate limiting to protect your application.

> Select the virtual server you want to modify.

**Don't:**
> The user can set up rate limiting to protect their application.

> The administrator selects the virtual server they want to modify.

## Notes

Avoid "we" when referring to F5 as the actor in instructions. Use "F5"
explicitly if you need to attribute a recommendation: "F5 recommends
setting up a health monitor for each pool."

Do not use "we" to mean the user and F5 together. It creates ambiguity.

Use gender-neutral language throughout. "They/their" is acceptable as a
singular pronoun when needed.
